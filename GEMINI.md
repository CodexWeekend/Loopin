# GEMINI.md

This file is a thin compatibility wrapper for Gemini-based agents. It is not a separate product brief.

## Start Here

Read these files in order:

1. `README.md`
2. `AGENTS.md`
3. `docs/implementation-status.md`
4. The canonical task docs you need

## Gemini-Specific Notes

- Treat `AGENTS.md` and the `docs/` canon as authoritative.
- Keep product scope, architecture, and current status in canonical repo docs only.
- If you make a decision that another contributor must inherit, record it in `docs/decision-log.md` or `docs/implementation-status.md`.
- If tooling on your machine differs from the original author environment, preserve the repo contract instead of creating a forked set of instructions in this wrapper.
